1 · The consolidated landscape

One curve per model, one point per effort level (low → max). X axis = relative cost, Y axis = relative quality — both computed identically : we never compare two raw benchmark scores across benchmarks (incompatible scales); within each benchmark we take the ratio between couples, normalise the benchmark to Opus 4.8 @medium = 1.0 (÷ the anchor when present, else a ×0.5-weighted bridge) and consolidate by source-weighted median. The oval = a robust, asymmetric uncertainty band (per-side spread with Huber ±1.5·MAD clipping : an outlier benchmark widens its side but doesn't dominate ; width = cost, height = quality). It is centred on the median (the dot is always inside), the quality axis is dilated near parity (1.0) to separate near-equal models, and every oval is faint until you hover it. Haiku 4.5 is not plotted here (see §2, §6 and the Pareto).

Both axes come from the same couple-atomic method : same-task ratios → per-benchmark normalisation → weighted-median consolidation. The effort→quality rise is therefore measured (CursorBench, OSWorld, Braintrust, zenn… sweep effort), not assumed; cost ladders are monotone (the quality dip at max on Fable / Sonnet 4.6 is the real over-thinking inverted-U). Bounds and gridlines recomputed dynamically.

Pareto frontier — the best intelligence per cost

The non-dominated (model, effort) couples : no other couple offers more quality at equal or lower cost. Below, the solid joined points form the frontier; faded points are dominated (you pay more without gaining capability). The Y axis is relative quality (same consolidated ratios as §1, anchor Opus 4.8 @medium = 1.0). Recomputed from the measured data :

How to read : left to right, each step on the frontier is the next "more intelligence" tier that is worth its extra cost. Anything not on it is dominated (another couple does at least as well for less). The faint grey curve is a smooth quadratic fit to the frontier (quality vs log-cost) — the reference envelope for the value score below.

Value score — distance to the frontier envelope

We fit a smooth curve to the Pareto frontier (quadratic in log-cost, R² ≈ 0.96, drawn faint grey on the chart above) — the expected quality for a given cost along the efficient edge. Each couple's score = its signed vertical distance to that curve, squashed by tanh(residual ⁄ RMS) : above the curve scores toward +1 (more quality than its cost buys on the edge), on the curve = 0, below toward −1. Frontier couples sit near the curve; the sign tells you whether a point pokes above the smooth edge (a genuine knee) or falls below it. Computed from the data.

Best value by task tier — what to actually run

Consolidating the frontier into a decision. We split tasks into four complexity tiers, each defined by the quality window it demands (relative quality, anchor Opus 4.8 @medium = 1.0). For each tier we pick, on the Pareto frontier and within that band, the couple with the best score/cost yield (quality ÷ cost) — the cheapest way to clear the bar. All picks and figures below are computed from the same data.

2 · The normalized matrix

Relative cost per model × effort (baseline Opus 4.8 @medium = 1.00), sorted by decreasing intelligence. These values are computed (not hand-entered) from the measured same-task ratios : within each benchmark we take the couple ratios, normalise to Opus 4.8 @medium, then each cell = the source-weighted median of the per-benchmark estimates (weight = number of sources; bridge ×0.5). Intensity = cost; the second figure is the robust CI (per-side Huber ±1.5·MAD, asymmetric ; wide = benchmarks disagree). To get back to dollars, multiply by a reference task cost (e.g. agentic ≈ $1.99/task for Opus 4.8 @max on the AA Index).

5 · What moves everything — the criteria, with examples

The matrix gives an order of magnitude. These four factors, each measured, modulate it strongly. Each card gives a real, quantified example.

Task type

Effort and model choice don't weigh the same depending on the nature of the work.

Code review — Haiku 4.5 scores an F1 of 0.365 at 1/3 the cost of Sonnet 4.6 (0.343). "Bigger isn't better" (arxiv 2606.15689).
Reasoning — Opus 4.5 medium→high : +29% score for 2.8× tokens (PostTrainBench). Here effort pays off.
Pure coding — raising the effort budget : +0.27 pt on SWE-Bench Pro (2606.17930). Plateau.

Complexity

The harder the task, the more effort costs — up to the point where it hurts.

192 stratified runs (whitekumalabo) : easy task ≈ $0.023 regardless of effort; hard task (L3) low→max = ×1.9.
Inverted U — Sonnet 4.6 on orchestration : low 0% / medium 16.7% / high 6.7% — high degrades (2606.08878).

Generation

Recent Opus models do more with less; Sonnet 5 is the verbose exception.

Opus 4.8 vs 4.7, same DB migration : 4.8 uses ~half the tokens, −40% cost (FullStack, 5 runs).
Sonnet 5 generates ~2.5× the tokens of Opus 4.8 on the AA Index → $2.29/task > $1.99 despite a lower per-token rate.

Cache & harness

Two invisible factors that can dominate both cost and score.

Cache — same code change, cache broken on Sonnet : $10.30 vs $3.83 for Opus (GitHub copilot #1770). Agentic work runs at 90–96% cache-read.
Harness — Opus 4.8 on Terminal-Bench : 84.6% (AA) vs 71.9% (Vals), same model. Confound #1.

6 · The no-thinking regime — the cheapest

Some sources run without extended thinking (reasoning off / raw API). This is not a point on the low→max sweep : it is a distinct cost regime, the floor. We treat it couple-atomically, exactly like §2 : each source that measures two models in the same regime on the same task gives a cost ratio; we take the median per pair. The per-model cost index (Haiku 4.5 = 1×) is chained along the measured ratios — indicative, not a decomposition. Fable 5 and Sonnet 5 have no published no-think run → absent here. Haiku 4.5 lives almost entirely in this regime (it doesn't expose the discrete effort parameter).

7 · Confidence & merge method

What is solid

  • HIGH Cross-model anchors @medium (÷Opus 4.8) : Fable ~2.1×, Sonnet 4.6 ~0.7×, Haiku 4.5 ~0.19× (no-think regime, §6) — several independent methods agree.
  • MEDIUM Cost effort slope low→max : ×1.9 (Sonnet 4.6) to ×4–5 (Opus 4.7 / Sonnet 5) — steeper on the agentic harness (CursorBench) than on computer-use (OSWorld).
  • MEDIUM Quality plateau + inverted-U on code/agentic (multi-paper).

What stays wide

  • WIDE Sonnet 5 : measured median ~0.7× Opus 4.8 (cheaper), but a verbose tail up to ~1.6× on long agentic work (AA) — hence the wide CI.
  • WIDE Opus 4.7 vs 4.8 : contradictory sources (effort/harness) → 0.6–1.4×.
  • WIDE xhigh/max tails and models with no measured effort sweep.

How the relative cost is merged — the procedure

  1. Same-task ratios only. We never compare two raw dollar figures from different sources (task sizes are incomparable). We keep only sources measuring ≥2 models/efforts on the same task: their ratio cancels the task-size variance. E.g. WorkBench gives Sonnet 4.6/Opus 4.8 = 0.58 on its tasks, whatever their absolute cost.
  2. The (model, effort) couple is atomic. Each node = one indivisible couple. We never assume that "effort" has the same effect across models : taking Haiku to max is not the same multiplier as taking Opus to max. No model×effort decomposition — separability would be a false assumption.
  3. Per-benchmark normalisation → weighted median. Inside each benchmark, every couple is divided by the anchor Opus 4.8 @medium when present, else bridged via its shared couples (down-weighted ×0.5). Each benchmark thus yields one normalised estimate per couple; a couple's value is the source-weighted median of those estimates. Benchmarks with a single couple are dropped (they would only echo the anchor). No path products, no cross-benchmark value comparison — only within-benchmark ratios.
  4. "Default" ≠ a single effort — verified source by source. We read each source's actual config (the Config ✓ column). Result : the "default" bucket is heterogeneous — some at max (CEO-Bench, EMB-Opus), high (SlopCode, OfficeQA), thinking off (STAGE-Claw, ianlpaterson), others unstated. Anthropic's docs explain why : the default depends on the model (Sonnet 5/Fable 5 always think; Opus 4.x & Sonnet 4.6 opt-in; Haiku & ≤4.5 have no such parameter). What licenses a ratio is the verified fact that each source applies the same config to all its models — not an assumed effort.
  5. Unlinked couple = flagged, never invented. A couple with no measured edge (Haiku @max — effort not exposed; or an effort sweep never published for that model) is not filled by a generic effort rule. One model's effort shape is never used as a template for another : it is either n/a, or estimated with an explicitly wide band.
  6. Confidence = agreement. The band is a per-side Huber spread (deviations clipped to ±1.5·MAD) : robust to an outlier benchmark yet still widened by it, and asymmetric. 3+ tight sources → narrow band (Sonnet 4.6 & Opus 4.8 low effort). Sources disagree → wide band (Fable max : Willison-SVG vs CursorBench; Sonnet 5, task-size dependent). Where nothing confirms, the band says so.
